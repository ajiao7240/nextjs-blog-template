# Kubernetes Usage Tips Brief

## Overview
- Topic: Practical Kubernetes usage tips for developers and SREs
- Purpose: Share actionable, real-world best practices
- Target Audience: DevOps engineers, backend developers, SREs
- Tone: Practical, concise, experience-based

## Key Points to Cover
1. Debugging common pod issues (CrashLoopBackOff, ImagePullBackOff)
2. Efficient resource management (requests/limits)
3. ConfigMap and Secret best practices
4. Using kubectl aliases and completion
5. Horizontal Pod Autoscaler tuning
6. Logging and monitoring patterns
7. Avoiding common security misconfigurations

## Requirements
- Must use real, verified techniques from production environments
- Must avoid theoretical or textbook advice
- Must include specific commands and yaml snippets
- Must reference official Kubernetes documentation where applicable
- Must be written in authentic, non-AI voice

## Exclusions
- Do not cover installation or cluster setup
- Do not explain basic concepts like pods or nodes
- Do not compare Kubernetes to other platforms
